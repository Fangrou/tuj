tParseName

tParseName Properties

Component Family
Data Quality
Function
This component retrieves names presented in different forms and extracts each item according to its type.
Purpose
The tParseName analyses a file containing names and extracts items according to their type, in order to improve the quality of the processed data. It also rejects the incorrect items.
Basic settings
Schema and Edit Schema
A schema is a row description, i.e., it defines the number of fields that will be processed and passed on to the next component. The schema is either built-in or remote in the Repository.
 
 
Built-in: You create the schema and store it locally for this component only. Related topic: How to set a built-in schema of Talend Integration Suite Studio User Guide.
 
 
Repository: You have already created the schema and stored it in the Repository. You can reuse it in various projects and job designs. Related topic: How to set a repository schema of Talend Integration Suite Studio User Guide.
 
Column to parse
Select the column from which you want to extract the different items.
Advanced settings
Ignore joint names. Mr John Smith and Ms Mary Jones -> Mr John Smith
Select this check box to ignore data containing joint names. This will make the data processing faster. Deselect it in order to return and cut this data by type.
 
Allow reversed. Smith, Mr AB -> Mr AB Smith
Select this check box so that names will be processed regardless of the form they are presented in. The component puts the names in order and performs the normal analysis. Note that if the name can be analyzed, its original order is not saved as a property.
 
Extend titles
Select this check box to use all possible titles. If you deselect it, you can only use "standard" titles.
Correct case
Select this check box in order to correct the case, that is to say, if needed, to put a capital letter at the beginning of a name.
 
Auto clean
Select this check box to delete the non-alphanumerical characters before trying to parse addresses.
 
tStatCatcher Statistics
Select this check box to collect log data for the job as a whole, as well as for each component.
Usage
This component acts as an intermediary. It requires an input and an output flow.
