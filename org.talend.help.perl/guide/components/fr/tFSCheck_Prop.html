tFSCheck


Ce composant n'est disponible dans la Palette que si vous avez souscrit à Talend Integration Suite MPx.

Propriétés du tFSCheck

Famille de composant
FileScale
Fonction
tFSCheck vérifie que tous les enregistrements d'un fichier d'entrée sont bien triés. Cette vérification peut être effectuée sur la totalité ou une partie des données. Ce composant peut vérifier très rapidement un fichier très volumineux. Pour optimiser les performances, le composant trie généralement les données avant de les traiter.
Objectif
tFSCheck permet de vérifier que les données du fichier source sont bien triées.
Basic settings
Schema type et Edit Schema
Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.
Cliquez sur Edit Schema pour modifier le schéma.

Si vous modifiez le schéma, il devient automatiquement built-in.

 
 
Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et jobs. Voir également : Paramétrer un schéma du Repository du Guide Utilisateur de Talend Integration Suite Studio.
 
 
Built-in : Le schéma est créé et conservé pour ce composant uniquement. Voir également : Paramétrer un schéma built-in du Guide Utilisateur de Talend Integration Suite Studio.
 
Property type
Peut être Built-in ou Repository.
 
 
Built-in : Propriétés utilisées ponctuellement.
 
 
Repository : Sélectionnez le fichier de propriétés du composant. Les champs suivants sont alors pré-remplis à l'aide des données collectées.
 
Input File Name
Nom du fichier contenant les données à vérifier.
 
Record separator (char)
Caractère, chaîne ou expression régulière séparant les lignes.
 
Field separator (char)
Caractère, chaîne ou expression régulière séparant les champs.
 
Header
Nombre de lignes d'en-tête à ignorer en début de fichier.
 
Footer
Nombre de lignes à ignorer en fin de fichier.
 
Criteria
Cliquez sur le bouton [+] pour ajouter autant de lignes que nécessaire afin de vérifier que le fichier source est bien trié.
Schema column : Cliquez dans la cellule et sélectionnez le nom de la colonne sur laquelle vous souhaitez vous baser pour trier le fichier

L'ordre est essentiel puisqu'il détermine la priorité de tri.

Sort type : Cliquez dans la cellule et sélectionnez le type de tri : numérique ou alphabétique.
Order type : Cliquez dans la cellule et sélectionnez l'ordre de tri : ascendant ou descendant.
Advanced settings
Generate FSLang File
Cochez cette case pour générer le fichier FSLang correspondant à votre Job et cliquez sur le bouton [...] à côté du champ FSLang File Name pour spécifier son chemin d'accès et son nom.
 
Assign FileScale Path
Cochez cette case puis cliquez sur le bouton [...] à côté du champ FileScale Path pour sélectionner le fichier exécutable correspondant au programme FileScale permettant d'exécuter le composant.
 
Specify Number of Process Child
Cochez cette case et saisissez le nombre de processus enfants à utiliser pour l'exécution de l'agrégation.
 
Custom FileScale Parameter (separated by ,)
Saisissez les paramètres de personnalisation, en les séparant par une virgule. Pour plus d'informations concernant les paramètres personnalisables, consultez l'annexe Langage de programmation FileScale (en anglais).
 
Check schema
Vérifie le schéma. Le résultat retourné dans la console est "Ok" si le schéma est valide, sinon, rien ne sera retourné.
 
tStatCatcher Statistics
Cochez cette case pour collecter les données de log au niveau du composant.
Utilisation
Ce composant traite des fichiers, ainsi il ne nécessite pas de flux de données en entrée ou en sortie. Il sert à vérifier les données contenues dans des fichiers volumineux.
Limitation
La limitation dépend des limites de la mémoire physique et de l'architecture du processeur. Par exemple, la taille totale des fichiers traités ne peut pas excéder la limite du système de fichiers pour supporter les fichiers très volumineux (maximum de la valeur 64 bits signée).
