Scénario 1 : Alimenter un entrepôt de données

Ce scénario Java suivant permet d'alimenter un entrepôt de données. Il est constitué de trois composants : le composant tMysqlInput correspond aux données de vos clients contenues dans la base Customers. Ces données sont modifiées à l'aide d'un composant tMap et les modifications sont transmises à la table Leadfact de votre entrepôt de données CRM à l'aide du composant de sortie tMysqlOutput.

    • Cliquez et déposez ces trois composants dans l'espace de modélisation et connectez-les à l'aide de lien de type Row Main.
    • Double-cliquez sur le composant tMysqlInput pour paramétrer ses propriétés.
    • Dans le champ Property Type, sélectionnez l'option Repository puis la connexion à la base de données Customers contenant les informations sur vos clients. Les informations de connexion seront renseignées automatiquement.



      • Si vous n'avez pas créé de connexion à votre base de données dans les métadonnées, sélectionnez Built-in dans le champ Property Type et renseignez les champs manuellement.

    • Dans le champ Schema, sélectionnez l'option Repository puis le schéma de la base Customers stocké dans les métadonnées.

Voir également : Paramétrer un schéma built-in et Paramétrer un schéma du Repository du Guide Utilisateur Talend Open Studio.

    • Dans le champ Table name, renseignez le nom de la table contenant les informations à modifier.
    • Cliquez sur le bouton Guess Query pour récupérer toutes les données de votre table.
    • Double-cliquez sur le composant tMap pour accéder au Map Editor. La zone input est déjà renseignée avec le schéma du composant d'entrée.
    • Sélectionnez les champs de la zone input et glissez-les jusqu'à la table Leadfact de la zone output. Pour plus d'informations concernant le mapping de données, consultez Mapper des flux de données.
    • Cliquez sur Ok pour valider.
    • Double-cliquez sur le composant tMysqlOutput pour paramétrer ses propriétés.
    • Dans le champ Property Type, sélectionnez l'option Repository puis la connexion à votre entrepôt de données CRM. Les informations de connexion seront renseignées automatiquement.



      • Si vous n'avez pas créé de connexion à votre entrepôt dans les métadonnées, sélectionnez Built-in et renseignez les champs manuellement.

Voir également : Paramétrer un schéma built-in et Paramétrer un schéma du Repository du Guide Utilisateur Talend Open Studio.

    • Dans le champ Table name, renseignez le nom de la table à alimenter, ici Leadfact.
    • Dans le champ Action on table, sélectionnez Create table pour créer la table.
    • Appuyez sur F6 pour créer et alimenter la table Leadfact de l'entrepôt de données CRM.

Scénario 2 : Extraire les modifications à l'aide du CDC

Ce scénario est basé sur le scénario précédent. Il alimente et modifie, au fil de l'eau, les données stockées dans l'entrepôt CRM et récupère et enregistre, toutes les nuits, ces modifications à l'aide de la fonction CDC dans une table dédiée. Et ces modifications sont ensuite extraites par les différents services intéressés : le service Marketing, le service Ventes et le service Finance.

Etape 1 : Configurer le CDC

Avant d'extraire les modifications apportées à votre entrepôt de données CRM, vous devez :

      • configurer la connexion à la base de données dédiée au CDC,
      • configurer la connexion à la base de données source et identifier la table à surveiller,
      • configurer la connexion entre le CDC et les données.

Pour cela :

      • Dans le Repository, sous le noeud Metadata, créez une connexion à votre base de données dédiée au CDC, ici CDC_connection.
      • Dans le Repository, sous le noeud Metadata, créez une connexion à votre entrepôt de données dont vous souhaitez extraire les changements, ici CRM_connection.
      • Cliquez-droit sur la connexion à votre CRM et sélectionnez l'option Retrieve schema pour récupérer le schéma de la table à surveiller.
      • Cliquez-droit sur le sous-dossier CDC Foundation de votre CRM et sélectionnez l'option Create CDC dans le menu contextuel.

La boîte de dialogue [Create change data capture] apparaît.

      • Dans le champ Set link Connection, sélectionnez la connexion CDC_connection.
      • Cliquez sur le bouton Create Subscriber. La boîte de dialogue [Create subscriber and Execute SQL Script] apparaît.
      • Cliquez sur Execute, puis sur Close.
      • Cliquez sur Finish pour valider la création de la table des souscripteurs.

Dans le sous-dossier CDC Foundation, la table des souscripteurs apparaît.

Vous devez précisez à quelle table les souscripteurs doivent souscrire puis activer leur souscription. Pour cela :

      • Cliquez-droit sur le schéma de la table Leadfact à surveiller dans votre entrepôt de données CRM et sélectionnez l'option add CDC dans le menu contextuel. La boîte de dialogue [Create Subscriber and Execute SQL Script] apparaît.
      • Dans les champs Events to catch, cochez les cases Insert, Update et Delete pour surveiller les insertions, mises à jours et suppression.
      • Dans le champ Subscriber Name, saisissez le nom du souscripteur ayant accès à ces modifications, ici Sub_Mktg pour le service Marketing.
      • Cliquez sur Execute, puis sur Close pour valider la souscription.

Dans le sous-dossier CDC Foundation, deux nouvelles tables apparaissent et le schéma de la table surveillée est marqué d'un symbole CDC vert.

Créez de nouveaux souscripteurs pour les services Finance et Ventes, respectivement Sub_Finance et Sub_Sales :

      • Cliquez-droit sur la table Leadfact par le CDC et sélectionnez l'option Edit CDC Subscribers. La boîte de dialogue [Edit CDC] apparaît.
      • Cliquez sur le bouton Add. La boîte de dialogue [Input subscriber name] apparaît.
      • Saisissez le nom du souscripteur, ici Sub_Finance et Sub_Sales.
      • Cliquez sur Execute puis Close pour valider la création.

Etape 2 : Modifier les données de votre CRM

Modifiez les données de vos clients dans votre CRM, par exemple, mettre tous les noms de vos clients en majuscule dans un soucis d'harmonisation.

      • Double-cliquez sur le composant tMap et saisissez row1.CustomerName.toUpperCase()devant la colonne CustomerName pour mettre tous les noms de vos clients en majuscule.
      • Cliquez sur Ok.
      • Double-cliquez sur le composant tMysqlOutput.
      • Dans le champ Action on table, sélectionnez None.
      • Dans le champ Action on data, sélectionnez Insert or update pour insérer ou mettre à jour les données de la table.
      • Appuyez sur F6 pour exécuter le Job de modification de données.

Pour visualiser toutes les modifications apportées, cliquez-droit sur la table Leadfact et sélectionnez l'option View All Changes.

Etape 3 : Extraire les modifications apportées aux données

Après avoir mis en place l'environnement CDC, vous pouvez créer un Job utilisant le composant MySQL CDC pour extraire les modifications apportées aux données de la table Leadfact. Pour cela:

      • Dans la Palette, cliquez-déposez les composants tMysqlCDC et tLogRow.
      • Reliez ces deux composants à l'aide d'une connexion de type Row Main.
      • Double-cliquez sur le composant tMysqlCDC pour paramétrer ses propriétés.
      • Dans le champ Property Type, sélectionnez l'option Repository puis la connexion CDC_connection à la base de données CDC. Les informations de connexion seront renseignées automatiquement.



        • Si vous n'avez pas créé de connexion à votre base de données de modifications dans les métadonnées, sélectionnez Built-in dans le champ Property Type et renseignez les champs manuellement.

      • Dans le champ Schema using CDC, sélectionnez l'option Repository puis le schéma de votre table Leadfact stockée dans les métadonnées.
      • Dans le champ Table using CDC, renseignez le nom de la table surveillée par le CDC.
      • Dans le champ Subscriber, renseignez le nom du souscripteur qui va extraire les modifications : Sub_Mktg pour le service Marketing, Sub_Sales pour le service Ventes ou Sub_Finance pour le service Finance.
      • Dans le champ Events to catch, cochez les cases correspondant au type de modifications que le souscripteur va extraire, ici cochez les trois cases pour les trois souscripteurs.
      • Double-cliquez sur le composant tLogRow pour paramétrer ses propriétés.
      • Cliquez sur le bouton Sync columns pour récupérer le schéma du composant précédent.
      • Enregistrez votre Job et appuyez sur F6 pour l'exécuter.

Le nom des clients est donc mis en majuscule et le type de modification apparaît ici sous la forme d'un U pour Update.

Une fois, ces modifications extraites, elles ne sont plus disponibles dans la table des modifications. Pour vérifier leur extraction, faites un clic-droit sur la table Leadfact surveillée par le CDC et sélectionnez l'option Views All Changes. Les modifications extraites n'y apparaissent plus.
