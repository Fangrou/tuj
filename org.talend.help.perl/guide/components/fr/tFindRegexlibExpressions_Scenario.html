Scénario : Se connecter à un service Web et retourner une liste d'expressions régulières

Ce scénario comporte un Job Java à trois composants, créé dans Talend Open Studio.

Ce scénario :

    • utilise le composant tFindRegexlibExpression pour se connecter à un serveur Web et collecter toutes les expressions régulières qui contiennent le mot "email" dans leur description,
    • utilise le composant tMap afin de réorganiser les données d'entrée dans le flux de sortie, et également de concaténer les deux champs du flux d'entrée en une seule colonne de sortie,
    • et écrit ensuite toutes les expressions collectées dans un fichier .csv.
    • Glissez les composants suivants de la Palette dans l'espace de modélisation graphique : tFindRegexlibExpressions, tMap, et tFileOutputDelimited.
    • Double-cliquez sur le composant tFindRegexlibExpressions afin d'ouvrir sa vue Basic settings et définir ses propriétés.

Le schéma de ce composant est en lecture seule et contient les champs suivants : Title, Expression, Description, Matches, Non-Matches, Author, et Rating (Titre, Expression, Description, Correspondances, Non-correspondances, Auteur, et Note).

    • Dans le champ Regexp Substring, définissez l'expression régulière que vous souhaitez utiliser comme filtre sur la liste des expressions régulières.
    • Dans le champ Key Words, définissez le(s) mot(s)-clé que vous souhaitez utiliser comme filtre sur la liste des expressions régulières.
    • Dans le champ Min Rate, définissez la note d'expression régulière que vous souhaitez utiliser comme filtre sur la liste des expressions régulières.

Dans ce scénario, l'objectif est de faire en sorte que le tFindRegexlibExpressions collecte toutes les expressions régulières du serveur Web ayant le mot "email" dans leur champ Description, et celles dont la note est de 1 minimum.

    • Connectez le tFindRegexlibExpressions et le tMap, à l'aide d'un lien Main row.
    • Double-cliquez sur le composant tMap afin d'ouvrir son éditeur et d'effectuer la réorganisation nécessaire des champs, ainsi que leur concaténation.
    • Dans l'éditeur du tMap, cliquez sur le bouton [+], en haut à droite, pour ouvrir une boîte de dialogue où vous pourrez nommer la nouvell table de sortie, regex dans ce scénario.
      Cela va créer un nouveau lien au tMap, portant le même nom, que vous allez utiliser pour connecter le tMap au composant suivant.
    • Dans le coin en bas à droite de l'éditeur, cliquez sur le bouton [+], afin de définir les champs de la table de sortie regex.
    • Dans la moitié supérieure de l'éditeur, glissez les champs de la table d'entrée afin de remplir les champs du schéma de sortie, comme nécessaire. Pour plus d'informations à propos du mapping de données, consultez le Guide Utilisateur Talend Open Studio (en anglais).

Dans ce scénario, l'objectif est de concaténer les champs Matches et Non-Matches du flux d'entrée en une seule colonne de sortie : Purpose (Objectif) et également d'obtenir une nouvelle colonne dans le schéma de sortie, appelée Path (chemin d'accès). Par ailleurs, le but n'est pas d'avoir d'informations sur les notes dans le schéma de sortie.

    • Cliquez sur OK pour valider et fermer l'éditeur du tMap.
    • Cliquez-droit sur le tMap, et sélectionnez le lien regex pour connecter le tMap au tFileOutputDelimited.
    • Double-cliquez sur le tFileOutputDelimited afin d'afficher ses Basic settings et définir ses propriétés.
    • Cliquez sur le bouton [...], à côté du champ File Name, pour sélectionner le fichier dans lequel vous souhaitez écrire les données de sortie.
    • Définissez les séparateurs de lignes et de champs, dans les champs respectifs.
    • Cochez la case Append si vous voulez ajouter les nouvelles lignes à la fin des enregistrements.
    • Cochez la case Include Header si vous souhaitez inclure les libellés des colonnes dans les données de sortie.
    • Au besoin, cliquez sur le bouton Edit schema pour voir les flux d'netrée et de sortie.
    • Sauvegardez votre Job et appuyez sur F6 afin de l'exécuter.
