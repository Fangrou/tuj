tThresholdViolationAlert

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l'édition correspondante de Talend Data Quality.
Ce composant n'est utilisable qu'à partir d'un Job ayant généré des rapports d'analyse.

Propriétés du tThresholdViolationAlert

Famille de composant
Data Quality
Fonction
Le tThresholdViolationAlert peut alerter en cas de violation de n'importe quel seuil parmi les seuils définis sur les indicateurs, lors d'analyses créées dans le studio de Talend Data Quality.
Objectif
Le tThresholdViolationAlert peut communiquer avec Talend Data Quality afin de vérifier les seuils définis sur les indicateurs utilisés dans des analyses créées dans Talend Data Quality. Vous pouvez donc l'utiliser avec d'autres composants, par exemple le tSendMail, afin d'alerter en cas de violation des seuils.
Pour plus d'informations sur comment générer un Job directement dans le studio de Talend Data Quality, qui alerte en cas de violation des seuils, consultez le Guide Utilisateur Talend Data Quality.
Basic settings
Schema et Edit Schema
Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.
 
 
Built-in : Le schéma est créé et conservé ponctuellement pour ce composant seulement. Voir également : Paramétrer un schéma built-in du Guide utilisateur de Talend Open Studio.
 
 
Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisable dans divers projets et Job designs. Voir également : Paramétrer un schéma du Repository du Guide utilisateur de Talend Open Studio.
 
Output Field
Sélectionnez dans la liste le champ de sortie qui contiendra les données d'entrée.
 
Addition Information
Champ automatiquement pré-rempli, correspondant au corps du message qui sera envoyé après l'analyse, s'il y a violation d'un seuil.
Advanced settings
Advanced separator (for number)
Cochez cette option pour modifier les séparateurs utilisés pour les nombres :
Thousands separator : définissez le séparateur utilisé pour les milliers.
Decimal separator : définissez le séparateur utilisé pour les décimaux.
 
Separator between Suffix and Field Value
Saisissez le caractère qui séparera le nom du champ de sa valeur.
 
Field separator
Caractère, chaîne ou expression régulière séparant les champs.
 
Row Separator
Chaîne (ex : "\n" sous Unix) séparant les lignes.
 
tStatCatcher Statistics
Cochez cette case pour collecter les métadonnées de process du Job, aussi bien au niveau du Job qu'au niveau de chaque composant.
Utilisation
Ce composant est un composant intermédiaire. Il requiert des composants d'entrée et de sortie. Le tThresholdViolationAlert ne fonctionne qu'avec une base de données MySQL.
