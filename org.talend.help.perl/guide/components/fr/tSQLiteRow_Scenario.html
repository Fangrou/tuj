Scénario : Mettre à jour des lignes SQLite

Le scénario suivant décrit un Job qui permet de mettre à jour un fichier de base de donnée SQLite en utilisant une requête prédéfinie et un fichier délimité.

    • Cliquez-déposez les composants tFileInputDelimited et tSQLiteRow.
    • Dans l'onglet Basic settings du composant tFileInputDelimited, parcourez les dossiers et sélectionnez le fichier d'entrée à utiliser pour mettre à jour les lignes dans la base de données.
    • Il n'y a ni en-tête ni pied de page. Le séparateur de ligne (Row separator) est un retour chariot et le séparateur de champ (Field separator) un double point.
    • Modifiez le schéma s'il n'est pas stocké dans le Repository.
    • Assurez-vous que la taille et le genre définissent correctement les colonnes.
    • Dans l'onglet Basic settings du composant tSQLiteRow, renseignez le champ Database avec le chemin d'accès au fichier à mettre à jour.
    • Le schéma est en mode read-only puisqu'il doit correspondre au schéma d'entrée.
    • Entrez la requête ou récupérez-la à partir du Repository. Dans le cas présent, on a mis à jour la colonne type_os en focntion de la valeur id définie dans le flux entrant. La requête se présente comme suit : 'Update download set type_os=? where id=?'
    • Cochez ensuite la case Prepared statement pour afficher la table des paramètres substituables.
    • Dans la table des paramètres d'entrée, ajoutez autant de lignes qu'il y a de paramètres substituables. Dans cet exemple, on veut définir type_os et id.
    • Renseignez le champ Commit every et choisissez le type d'encodage à partir de la liste déroulant Encoding type.
    • Enregistrez le Job et appuyez sur F6 pour l'exécuter.

La table dowload table de la base de données SQLite est ainsi mise à jour le code avec le nouveau code type_os et selon le fichier délimité d'entrée.
