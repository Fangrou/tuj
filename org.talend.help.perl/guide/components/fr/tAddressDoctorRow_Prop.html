tAddressDoctorRow

This component does not appear in the Palette of the studio unless you have the right subscription to the right edition of Talend Data Quality.

The address management component discussed here is the result of Talend collaboration with AddressDoctor, one of the world leaders for global address data quality.

For more information about the enterprise and its software tools, visit http://www.addressdoctor.com.

tAddressDoctorRow properties

Component family
Data quality
Function
tAdressDoctorRow helps to capture, validate, and cleanse addresses for you. It does all the dirty work of address matching, correcting spelling, formatting and so on.
Purpose
tAdressDoctorRow validates and corrects addresses against databases coming from postal companies around the world.
Basic settings
Schema type and Edit Schema
A schema is a row description, i.e., it defines the number of fields that will be processed and passed on to the next component. The schema is either built-in or remote in the Repository.
 
Built-in: You create the schema and store it locally for this component only. Related topic: Setting a built-in schema of Talend Open Studio User Guide.
 
Repository: You have already created the schema and stored it in the Repository. You can reuse it in various projects and job designs. Related topic: Setting a Repository schema of Talend Open Studio User Guide.
 
User Id and password
Enter your user ID and password.
To get a user ID and a password, you need first to sign up for an AddressDoctor Data Quality Center account at https://www.addressdoctor.com/AcctMngr/newaccount.aspx?lang=en. Then you will receive your User Id and Password by email. With this register-for-free operation, you can process limited number of addresses. If you need to process a higher number of addresses, you can purchase transactions online at .https://www.addressdoctor.com
 
Address Column
Select the address column you want to validate and clean.
 
Origin Country
Select the origin country from the list.
This option controls the output of the formatted address. Whenever mail or other items are sent cross border, it is necessary to place the country of destination in the address. Parameters for this option are:
ALWAYS_USE_DESTINATION_COUNTRY (by default): The destination country will always be used in the formatted address
NEVER_USE_DESTINATION_COUNTRY:The destination country will never be used in the formatted address
...
Select the country of origin XXX where XXX is equal to the ISO3 country code. e.g.: AUS for Australia, JPN for Japan etc.
 
Country Type
Select the country type from the list.
This option controls the way the country name is returned in the formatted address. Parameters for this option are:
ISO_2: Returns the country as an ISO2 country code.
ISO_3: Returns the country as an ISO3 country code.
ISO_NUMBER: Returns the country as a numeric ISO code.
ABBREVIATION: Returns the country abbreviation as often used in car license plates.
NAME_CN: Returns the name in Chinese.
NAME_DA: Returns the name in Danish.
...
 
Line Separator
Select a line separator from the list.
The line separation character must be set when passing or retrieving data that contain multiple lines. Parameters for this option are:
LF: Uses a linefeed as the line separation character. You can also use this value when you would prefer a carriage return linefeed.The carriage return linefeed is not supported for technical reasons.
SEMICOLON: Uses a semicolon as the line separation character.
COMMA: Uses a comma as the line separation character
TAB: Uses a TAB character as the line separation character.
NO_SEPARATOR: Uses no special line separation character. Fields will only be separated by a space.
 
Capitalize
Select from the list the letter case of the written form of the cleaned address.
Usage
This component is an intermediary step. It requires an input flow as well as an output.
