Scénario : Vérifier les données dans des colonnes multiples par rapport aux modèles donnés

Ce scénario Java décrit un Job comprenant quatre composants, qui vérifie le code des clients, leur code postal et leur adresse e-mail par rapport aux modèles donnés. Il liste les données correspondant au modèle sélectionné, et celles qui ne correspondent pas.

Les résultats de la vérification sont écrits dans deux fichiers de sortie : le premier pour les valeurs correspondant aux modèles sélectionnés, et le second pour les valeurs qui ne correspondent pas aux modèles sélectionnés.

Dans ce scénario, vous avez déjà stocké les principaux schémas d'entrée dans le Repository. Pour plus d'informations à propos du stockage de métadonnées de schéma dans le Repository, consultez Configurer un schéma File Excel du Guide Utilisateur Talend Open Studio.

La table principale d'entrée contient six colonnes : ref_code, ref_name, ref_address, ref_zipcode, ref_city, et ref_email. Vous allez vérifier les entrées dans les colonnes ref_code, ref_zipcode et ref_email par rapport aux modèles donnés.

    • Dans la vue Repository, cliquez sur Metadata > DB Connections où vous avez stocké les principaux schémas d'entrée, et glissez le fichier correspondant dans l'espace de modélisation graphique.
      La boîte de dialogue [Components] s'ouvre alors, avec le composant correspondant, sélectionné par défaut.
    • Cliquez sur OK pour déposer un composant tMysqlInput dans l'espace de modélisation graphique.
      La table d'entrée utilisée dans ce scénario est appelée customer. Elle contient plusieurs colonnes, dont les trois colonnes ref_code, ref_zipcode et ref_email par rapport auxquelles vous voulez vérifier le modèle.
    • Glissez un tMultiPatternCheck de la Palette dans l'espace de modélisation graphique, ainsi que deux tLogRow.
    • Connectez le composant d'entrée principal au tMultiPatternCheck à l'aide d'un lien Row Main.
    • Connectez le tMultiPatternCheck aux deux composants tLogRow à l'aide des liens Matches et Non Matches.
    • Double-cliquez sur le composant de base de données d'entrée afin d'afficher sa vue Component.

Les champs des propriétés du tMysqlInput sont automatiquement remplis. Si vous n'avez pas stocké localement votre schéma d'entrée dans le Repository, remplissez les informations manuellement, après avoir sélectionné Built-in dans les listes Schema et Property Type.

    • Double-cliquez sur le composant tMultiPatternCheck afin d'afficher sa vue Component et définir ses propriétés.
    • Cliquez sur le bouton Edit schema pour ouvrir une boîte de dialogue. Vous pouvez définir les données que vous souhaitez passer aux composants de sortie.

Dans cet exemple, l'objectif est de passer au composant tMultiPatternCheck toutes les colonnes d'entrée.

    • Cliquez sur OK pour fermer la boîte de dialogue et procéder à l'étape suivante.
    • Cliquez sur la colonne Check Pattern et sélectionnez dans la liste le modèle par rapport auquel vérifier les données de la colonne ref_code, Numeric dans cet exemple.
    • Répétez l'opération afin de sélectionner les modèles pour les colonnes ref_zipcode et ref_email.
      Dans cet exemple, l'objectif est de vérifier que toutes les données de la colonne ref_zipcode sont des codes postaux français valides, et que les adresses e-mail des clients de la colonne ref_email sont des adresses valides.
    • Dans la colonne Is Case Sensitive, cochez les cases en face des colonnes sur lesquelles vous voulez prendre en compte la casse durant la vérification des modèles.
    • Dans la colonne Check, cochez les cases en face des colonnes que vous voulez vérifier par rapport aux modèles définis, ref_code, ref_zipcode et ref_email dans cet exemple.
    • Double-cliquez sur le premier composant tLogRow afin d'afficher sa vue Component et définir ses propriétés.
    • Dans la zone Mode, sélectionnez l'option Table pour afficher les résultats sous forme de tableau.
    • Répétez l'opération pour le second composant tLogRow.
    • Sauvegardez votre Job et appuyez sur F6 afin de l'exécuter.

Deux tables de sorties s'affichent dans la console. La première liste les données d'entrée dans les trois colonnes définies, qui correspondent au modèle donné. La seconde liste les entrées qui ne correspondent pas aux modèles sélectionnés.
