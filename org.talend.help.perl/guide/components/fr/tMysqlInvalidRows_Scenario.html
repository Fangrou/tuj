Scénario : Lire des adresses e-mail d'une table d'une base de données, et extraire les lignes invalides

Ce scénario Java est un Job à deux composants créé dans Talend Open Studio. Dans ce Job, le tMySQLInvalidRows lit la colonne des adresses e-mail d'une table d'une base de données MySQL, vérifie la valeur de la colonne des e-mails, par rapport aux modèles de Talend Data Quality donnés, et extrait les lignes invalides, puis les affiche dans la console.

Ci-dessous est la table de la base de données que vous utilisez dans l'exemple, la colonne Email contient des données invalides qui doivent être corrigées.

Dans ce scénario, vous avez déjà stocké les schémas de la table d'entrée dans le Repository. Pour plus d'informations sur le stockage des schémas de métadonnées dans l'arborescence du Repository, consultez le Guide Utilisateur de Talend Open Studio (en anglais).

    • Dans l'arborescence du Repository, développez les noeuds Metadata et Db Connections où vous avez stocké les schémas d'input, puis glissez la métadonnée de connexion correspondant dans l'espace de modélisation graphique.
      La boîte de dialogue [Components] s'ouvre.
    • Sélectionnez tMySQLInvalidRows dans la liste, et cliquez sur OK pour fermer la boîte de dialogue.
      Le composant tMySQLInvalidRows apparaît dans l'espace de modélisation graphique. La table MySQL utilisée dans ce scénario est appelée employees. Elle contient l'ID, FirstName, Initial et Email (ID, Prénom, Initiales et E-mail) des employés.
    • Glissez le tLogRow de la Palette dans l'espace de modélisation graphique.
    • Connectez les deux composants à l'aide d'un lien Main Row.
    • .Double-cliquez sur le tMySQLInvalid Rows afin d'afficher sa vue Basic settings et définir ses propriétés.

Tous les champs relatifs à la connexion sont automatiquement renseignés. Si vous ne définissez pas vos fichiers d'entrée dans le Repository, remplissez les champs manuellement, après avoir sélectionné Built-in dans le champ Property Type.

    • Dans la liste Analyzed column, sélectionnez la colonne que vous souhaitez analyser, Email dans cet exemple.
    • Dans la liste Patterns list, sélectionnez le modèle de base de données Talend Data Quality par rapport auquel vous voulez vérifier la colonne Email, Email Address dans cet exemple.



      • Si vous cochez la case Custom pattern, vous pouvez personnaliser l'expression régluière par rapport à laquelle vos données seront vérifiées.

    • Double-cliquez sur le tLogRow afin d'ouvrir sa vue Basic settings et définir ses propriétés comme nécessaire.
    • Sauvegardez votre Job et appuyez sur F6 pour l'exécuter.

Le composant tMySQLInvalidRows a analysé la colonne Email de la table MySQL en la comparant avec le modèle SQL sélectionné (Email Address), puis a extrait toutes les données invalides devant être corrigées.
