Scénario : Agréger les données d'un fichier volumineux

Ce scénario a été créé en Java et décrit un Job agrégant, très rapidemement, un fichier volumineux contenant des données clients en fonction de l'Etat dans lequel ils sont basés, calculant le nombre de clients ayant été agrégés pour chaque Etat et faisant une moyenne de leurs bénéfices grâce au composant tFSAggregate.

Assurez-vous d'avoir bien enregistré localement le fichier exécutable correspondant à votre système d'exploitation afin de pouvoir indiquer son chemin d'accès dans Talend Integration Suite Studio.

Dans ce scénario, vous avez déjà centralisé le schéma du fichier dans le Repository. Pour plus d'informations concernant la création et l'utilisation de métadonnées de schémas dans le Repository, consultez les sections Configurer un schéma File Delimited et Déposer des composants à partir du répertoire Metadata du Guide Utilisateur de Talend Integration Suite Studio.

Le fichier contient neuf colonnes : id, CustomerName, CustomerAddress, idState, id2, RegTime, RegisterTime, Sum1 et Sum2.

    • Dans le Repository, développez le noeud Metadata et le noeud de la métadonnée que vous venez de créer contenant le schéma d'entrée et glissez ce noeud dans le Job designer.
      La boîte de dialogue [Components] apparaît.
    • Sélectionnez le composant tFSAggregate dans la liste et cliquez sur OK pour fermer la boîte de dialogue.
      Le composant tFSAggregate apparaît dans le Job designer.
    • Double-cliquez sur le composant tFSAggregate pour paramétrer ses propriétés dans l'onglet Basic settings de la vue Component.

Tous les champs de propriétés du tFSAggregate sont automatiquement renseignés. Si vous n'avez pas défini le schéma dans les métadonnées du Repository, renseignez manuellement ces informations après avoir sélectionné le mode Built-in dans les listes Schema Type et Property Type.

    • Dans le champ Output File Name, indiquez le nom et le chemin d'accès du fichier de sortie contenant les données agrégées.
    • Dans la zone Group by, cochez la case correspondant à la colonne sur laquelle effectuer l'opération de regroupement. Vous pouvez sélectionner plusieurs colonnes afin d'effectuer des regroupements sur plusieurs critères. Pour ce scénario, regroupez les données de la colonne idState.
    • Dans la zone Operations, cliquez sur le bouton (+) pour ajouter deux colonnes contenant le résultat des opérations d'agrégation.
    • Dans le champ Additional Output column de la première ligne, saisissez un nom pour la première colonne supplémentaire, count pour ce scénario.
    • Dans le champ Function, sélectionnez l'opération d'agrégation que vous souhaitez exécuter, count pour ce scénario.
    • Dans le champ Input Column, sélectionnez la colonne dont la valeur d'entrée sera récupérée pour l'opération d'agrégation, id pour ce scénario.

Ainsi, la colonne count sera ajoutée au fichier de sortie et contiendra le nombre d'id des clients regroupés par Etat.

    • Dans le champ Additional Output column de la deuxième ligne, saisissez le nom de la deuxième colonne supplémentaire, avg.
    • Dans le champ Function, sélectionnez l'opération d'agrégation que vous souhaitez exécuter, avg pour ce scénario.
    • Dans le champ Input Column, sélectionnez la colonne dont la valeur d'entrée sera récupérée pour l'opération d'agrégation, Sum1 pour ce scénario.

Ainsi, la colonne avg sera ajoutée au fichier de sortie et fera la moyenne des Sum1 des clients par Etat.

    • Cliquez sur l'onglet Advanced settings pour paramétrer les propriétés avancées du composant et cochez la case Assign FileScale Path et dans le champ FileScale Path, indiquez le chemin d'accès au fichier exécutable correspondant à votre système d'exploitation.
    • Enregistrez le Job et appuyez sur F6 pour l'exécuter.

Une barre de progression apparaît sous le composant tFSAggregate dans le Job designer pour montrer le pourcentage de progression de l'opération d'agrégation. Cette barre de progression vous montrera la rapidité de l'exécution de l'agrégation.

Lorsque le pourcentage de progression atteint les 100%, les données sélectionnées sont regroupées et transmises en sortie, ainsi que les deux nouvelles colonnes contenant le compte des id des données regoupées et la moyenne des Sum1.
